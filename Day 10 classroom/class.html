HTML HTML stands for Hyper Text Markup Language .
HTML is the standard markup language for creating Web pages .
HTML describes the structure of a Web page.
HTML consists of a series of elements.
HTML elements tell the browser how to display the content.
Markup A markup language is one that is designed for defining and presenting text.
HTML (HyperText Markup Language), is an example of a markup language.
Tag The html tag represents the root of an HTML document.
Types of tags 1.Container tag(or)pair tags
2. Empty tags(or)void tags (or)self closing tags
Element In HTML, an element is a section of an HTML document.
Some HTML elements represent visible components on a web page,
such as text, images, or buttons, while others denote different sections of the page
or provide meta-information about the document.
Basic Structure of HTML HTML documents have a fundamental structure that consists of several essential elements:
Document Declaration Type DOCTYPE html: This declaration is placed at the very top of an HTML document.
Root Element html: This is the root element of an HTML document.
It encompasses all the content and other HTML elements.
Head Section > head: This section contains meta-information about
the document, such as its title, character set, and links to external resources like stylesheets and scripts.
>title: Sets the title of the document, which appears in the browser tab.
> meta: Provides metadata such as the character set, author, and viewport settings.
> link: Links to external resources like CSS files.
Heading tags in html Heading tags in HTML are used to define headings or titles for sections of content on a webpage.
>h1 Largest heading
>h2: Second largest heading
>h3: Third largest heading
>h4: Fouth largest heading
>h5: fifth largest or second smallest heading
>h6: Smallest heading
Preformatted Tag >pre: This tag is used for preformatted text. It preserves spaces, line breaks, and formatting as they are in the HTML code.
Paragraph tags >p: This tag is used to define a paragraph. It is a block-level element and automatically adds some space before and after the content.
Comments comments are used to insert notes or annotations within the code that are ignored by the web browser when rendering the webpage.
Tables HTML tables are used to display data in a structured format with rows and columns.
They are defined using the table tag.
Basic Structure A basic HTML table consists of:
table: The container for the table.
tr (table row): Defines a row in the table.
th (table header): Defines a header cell in the table.
td (table data): Defines a standard cell in the table.
Attributes border Defines the border width of the table and its cells.
cellpadding Defines the space between the cell content and its border.
cellspacing Defines the space between cells.
Width and height Defines the width and height of the table.
bgcolor Defines the background color of the table, rows, or cells.
Table Sections thead> Groups the header content in a table.
tbody> Groups the body content in a table.
tfoot> Groups the footer content in a table.
Caption A caption tag is used to provide a title for the table.

FORMATTING TAGS It is used to define the visual presentation of the web content.
some notes about formatting tags in HTML: 1.Use of b> and i> Tags: b> and i> tags were traditionally used for bold and italic formatting, respectively.
2. strong> and em> Tags: strong> and em> tags are used to indicate semantic importance and emphasis, respectively.
3. u> Tag: The u> tag is used to underline text. However, it's generally not recommended because underlined text is commonly associated with hyperlinks.
4. del> Indicates deleted text, typically rendered with a strikethrough
5. mark>/mark> Highlights text, often used to mark text as important
6. sup> and sub> Tags: sup> and sub> tags are used to display superscript and subscript text, respectively.
7. abbr> tags are used to define abbreviations, respectively.
8. blockquote> Tag: Defines a section that is quoted from another source
Structure of url A URL consists of ten parts: the scheme, subdomain, top-level domain, second-level domain, subdirectory, parameter, port, path, query, and fragment
Protocol protocol is a set of rules and standards that define how data is transmitted and received over a network
HTTP (Hypertext Transfer Protocol) Purpose: The foundation of data communication on the web.
Function: Defines how messages are formatted and transmitted, and how web servers and browsers should respond to various commands.
Port: Typically uses port 80.
Stateless Protocol: Each request from a client to a server is independent.
HTTPS (Hypertext Transfer Protocol Secure) Purpose: Secure version of HTTP.
Function: Uses encryption to secure data between the client and server, providing confidentiality and integrity.
Port: Typically uses port 443.
Encryption: Uses SSL/TLS protocols for encryption.
Subdomain A subdomain is an additional part of your main domain name.
It helps organize and navigate to different sections of your website.
Second-Level Domain (SLD): The part directly to the left of the TLD.
Chosen by the domain owner.
Example: example in www.example.com
Domain A domain name is a unique address used to identify a website on the internet.
Domain Name System (DNS) The Domain Name System (DNS) is a hierarchical system that translates human-readable domain names into IP addresses.
TLD TLDs are a fundamental component of the domain name system, playing a crucial role in organizing and identifying websites on the internet.
Popular TLDs and Their Uses .com: Commercial entities, the most popular and widely recognized TLD.
.org: Non-profit organizations, open to anyone.
.net: Originally for network services, now open to anyone.
.edu: Educational institutions, restricted to accredited entities.
.gov: U.S. government entities, restricted.
.in: indian websites
Path path specifies the location of a resource on the web server.
Absolute Paths An absolute path specifies the complete URL to a resource.
This means it includes the protocol (such as http:// or https://), the domain name, and the full path to the resource.
Relative Paths A relative path specifies the location of a resource in relation to the current document.
Relative paths are shorter and more flexible than absolute paths.
Anchor tags Anchor tags, are used in HTML to create hyperlinks
1. Basic Structure: An anchor tag is written as a> /a>
2.Href Attribute: The most important attribute of an anchor tag is href, which stands for "hypertext reference".
3.Link Text: The text between the opening and closing a> tags is what the user sees and clicks on.
4.Target Attribute: The target attribute specifies where to open the linked document.
